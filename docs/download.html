Install PaperSeek

Pick your browser and follow the steps. Takes under a minute.

Google Chrome
Silent PDF download via Chrome Web Store
Add to Chrome
1
Click "Add to Chrome" aboveYou'll be taken to the PaperSeek listing on the Chrome Web Store.
2
Confirm the installClick "Add extension" in the confirmation dialog that appears.
3
Open DeepSeekNavigate to chat.deepseek.com — the PaperSeek toolbar appears automatically at the bottom-right.

Silent auto-download: On Chrome, clicking Export PDF silently downloads the file to your default downloads folder — no print dialog, no extra steps.

Mozilla Firefox
PDF via system print dialog
Add to Firefox
1
Click "Add to Firefox" aboveThis takes you to the Firefox Add-ons listing.
2
Confirm the installClick "Add" in the Firefox permission dialog.
3
Open DeepSeek and exportWhen you click Export PDF, Firefox opens the system print dialog. Choose Save as PDF as the destination.

Print dialog tip: In Firefox's print dialog, set Destination to "Save to PDF", choose your paper size and orientation, then click Save.

Safari
Load unpacked — print dialog for PDF
Download source
1
Enable Safari extension developmentOpen Safari → Preferences → Advanced → check Show Develop menu.
2
Download the sourceClick "Download source" above and unzip the folder somewhere permanent on your Mac.
3
Convert with XcodeRun xcrun safari-web-extension-converter /path/to/deepseek-exporter in Terminal. Xcode will open with a generated project.
4
Build and allow unsigned extensionsIn Xcode, press Run. Then in Safari → Develop → Allow Unsigned Extensions, and enable PaperSeek.

Safari requires re-enabling unsigned extensions each time you restart Safari. For a permanent install, you'd need an Apple Developer account to sign the extension.

Common questions

Does PaperSeek read my chat messages?
No. The extension reads your chat DOM locally to select elements for printing. Nothing is uploaded or sent to any server — not our servers, not DeepSeek's, not anyone else's.
Why does it need the "debugger" permission?
Chrome's Page.printToPDF command — the only way to silently download a full page as a high-quality PDF without a print dialog — requires attaching Chrome's debugger to the tab. It's attached only during the export, and detached immediately after. No other debugger functionality is used.
Why does Firefox show a print dialog?
Firefox doesn't expose the Chrome DevTools Protocol (Page.printToPDF), so we fall back to window.print(). In the dialog, set Destination to "Save to PDF" to get your file. The toolbar and UI chrome are hidden automatically during printing.
Is it free? Will it stay free?
Yes, and yes. PaperSeek is fully open-source under MIT. There's no subscription, no premium tier, no ads. Fork it, modify it, use it forever.
The toolbar isn't showing up. What do I do?
Make sure you're on chat.deepseek.com (not a different subdomain). Try a hard refresh (Ctrl+Shift+R). If the extension was just installed, navigate away and back to the chat page. If it still doesn't appear, open an issue on GitHub.
